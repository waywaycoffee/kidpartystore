# 管理后台使用指南

## 📋 概述

已创建类似 Shopify 的管理后台系统，包含以下核心功能：

1. **仪表板** - 销售数据概览和快速操作
2. **产品管理** - 产品的增删改查
3. **图片管理** - 图片上传和管理
4. **销售分析** - 数据分析和图表展示

---

## 🚀 快速开始

### 1. 安装新依赖

```bash
npm install
```

新增依赖：
- `recharts` - 图表库（用于数据分析）
- `react-dropzone` - 图片拖拽上传

### 2. 访问管理后台

启动开发服务器后，访问：
```
http://localhost:3000/admin
```

---

## 📁 功能模块

### 1. 仪表板 (`/admin`)

**功能：**
- 性能指标展示（访问量、销售额、订单数、转化率）
- 时间范围筛选（7天、30天、90天）
- 渠道筛选（所有渠道、在线商店、POS）
- 快速操作卡片（产品管理、在线商店自定义）
- 设置任务提醒（支付、运费、URL重定向）
- 销售趋势图表

**使用说明：**
1. 访问 `/admin` 查看仪表板
2. 使用顶部下拉菜单选择时间范围和渠道
3. 点击快速操作卡片进行相应操作

---

### 2. 产品管理 (`/admin/products`)

**功能：**
- 产品列表展示（表格形式）
- 产品搜索和筛选
- 批量操作（选择多个产品）
- 添加新产品
- 编辑产品
- 删除产品

**使用说明：**

#### 添加产品：
1. 点击 "添加产品" 按钮
2. 填写产品信息：
   - 基本信息（名称、价格、描述、图片）
   - 分类和主题
   - 产品属性（年龄、尺寸、材质、认证）
   - 库存和状态
3. 点击 "保存产品"

#### 编辑产品：
1. 在产品列表中点击 "编辑"
2. 修改产品信息
3. 保存更改

#### 删除产品：
1. 在产品列表中点击 "删除"
2. 确认删除操作

#### 批量操作：
1. 勾选多个产品
2. 使用批量操作按钮（批量发布、批量删除）

---

### 3. 图片管理 (`/admin/images`)

**功能：**
- 拖拽上传图片
- 图片预览
- 图片分类（产品图片、横幅、Logo、其他）
- 设置网站横幅和Logo
- 删除图片

**使用说明：**

#### 上传图片：
1. 选择图片类型（产品图片、横幅、Logo等）
2. 拖拽图片到上传区域，或点击选择文件
3. 支持格式：PNG, JPG, JPEG, GIF, WEBP

#### 设置网站图片：
1. 上传或选择图片
2. 点击 "设为横幅" 或 "设为Logo"
3. 图片将保存到 localStorage（实际项目中应保存到服务器）

#### 删除图片：
1. 鼠标悬停在图片上
2. 点击 "删除" 按钮
3. 确认删除

---

### 4. 销售分析 (`/admin/analytics`)

**功能：**
- 销售趋势图表（折线图）
- 产品销售排行（柱状图）
- 产品销售占比（饼图）
- 详细数据表格

**使用说明：**
1. 访问 `/admin/analytics`
2. 选择时间范围（7天、30天、90天、一年）
3. 查看各种图表和数据表格

---

## 🔧 API 接口

### 统计数据
```
GET /api/admin/stats
```
返回仪表板统计数据

### 产品管理
```
GET /api/admin/products          # 获取产品列表
POST /api/admin/products         # 创建产品
GET /api/admin/products/[id]     # 获取产品详情
PUT /api/admin/products/[id]     # 更新产品
DELETE /api/admin/products/[id]  # 删除产品
```

### 分析数据
```
GET /api/admin/analytics?range=30
```
返回销售分析数据

## 💾 数据存储

当前使用 JSON 文件存储数据（位于 `data/` 目录）：

- `data/products.json` - 产品数据
- `data/orders.json` - 订单数据（用于分析）

**注意：**
- 数据文件已添加到 `.gitignore`，不会提交到 Git
- 生产环境建议使用数据库（MongoDB、PostgreSQL等）

## 🔒 安全建议

### 当前实现（开发环境）：
- 无身份验证（所有用户可访问）
- 数据存储在本地 JSON 文件

### 生产环境建议：
1. **添加身份验证**
   - 使用 NextAuth.js 或类似方案
   - 实现登录/登出功能
   - 角色权限管理

2. **数据安全**
   - 使用数据库（MongoDB、PostgreSQL）
   - 数据加密
   - 定期备份

3. **API 安全**
   - 添加 API 密钥验证
   - 速率限制（Rate Limiting）
   - 输入验证和清理

4. **图片存储**
   - 使用云存储（AWS S3、Cloudinary）
   - 图片压缩和优化
   - CDN 加速

## 🐛 常见问题

### Q: 数据丢失了？
A: 检查 `data/` 目录是否存在，JSON 文件是否完整。数据存储在本地文件系统中。

### Q: 图片上传失败？
A: 当前实现使用浏览器预览，实际项目中需要配置图片上传 API 和存储服务。

### Q: 图表不显示？
A: 确保已安装 `recharts` 依赖：`npm install recharts`

### Q: 如何添加更多分析功能？
A: 修改 `/app/api/admin/analytics/route.ts` 添加更多数据计算逻辑。
